2.Illustrate the differences between these classifiers and compare the model performances.¶

2.1 classifiers model performances¶

2.1.1 Decision Tree Classifier

The accuracy of the training set and validation set is very high, reaching 0.997558 and 0.996948 respectively, indicating that the overall classification ability is strong. The accuracy of training set is 1.000000, but the accuracy of validation set is 0.993711, which shows a certain decline and overfitting. The recall is 0.837288 in training set and 0.802030 in validation set, which indicates that there is room for improvement in recognition ability of positive samples. The F-score is 0.911439 in training set and 0.887640 in validation set, which shows good comprehensive performance. But obviously there's an overfitting problem. Decision tree can automatically learn the relationship between features from the data set, has certain adaptability to the feature classification of the data set, and can construct reasonable classification rules. It does not require complex preprocessing of the data and can directly process the raw data, on which training and classification can be quickly performed. But decision tree is easy to overfit, which leads to the decrease of accuracy of validation set.

2.1.2 Random Forest

The random forest's indicators are almost the same as the decision tree, and the accuracy, precision, recall, and F-score of the training set and validation set are consistent with the decision tree. The confusion matrix is similar, probably because the base classifier of the random forest is a decision tree, and the random forest does not exert a clear advantage over the decision tree at the current parameter settings. Random forests can theoretically reduce the risk of overfitting by integrating multiple decision trees. However, in this dataset, this advantage is not fully reflected due to the characteristics of the data itself.

2.1.3 Bagging

The accuracy of the training set and validation set is very high, 0.999491 and 0.999695 respectively. The accuracy, recall and F-score are excellent on the training set and validation set, indicating strong generalization ability. Confusion matrix shows that positive and negative samples are identified very accurately, and the number of false positive and false negative examples is very small. Bagging trains multiple base classifiers by sampling the original dataset with replacement, reduces the variance of the base classifier, and improves the stability and generalization ability of the model. For this dataset, Bagging effectively reduces the risk of overfitting the model, allowing the model to maintain good performance on the validation set. In this data, decision tree overfitting as a base classifier is more serious, so it also affects the performance of bagging.

2.1.4 Ada Boost

The accuracy, precision, recall, and F-score of both the training and validation sets are high, and the overall performance is excellent. The difference between training set and validation set is small, which indicates that the overfitting phenomenon is not serious. The confusion matrix shows fewer misclassification errors for positive and negative samples. AdaBoost gradually improves the performance of the model by iteratively training weak classifiers and adjusting the sample weights according to the classification error rate, so that subsequent weak classifiers pay more attention to the previous classification error samples.

2.1.5 KNeighborsClassifier

The accuracy of the training set and validation set is also high, 0.998983 and 0.998627, respectively. The precision of training set is 0.996390, and the precision of validation set is 1.000000, but the recall of training set and validation set is 0.935593 and 0.908629 respectively, which is slightly lower than other classifiers. The F-score was 0.965035 in the training set and 0.952128 in the validation set. The confusion matrix shows that false positive examples are 0, but false negative examples are 18, which indicates that there are some problems in identifying positive samples. KNeighbor Classifier is based on nearest neighbor principle, which does not need complicated training process and can deal with nonlinear data. On this data set, we can classify samples according to their similarity, and get a good accuracy. The computational complexity is high, and the distance between the sample to be classified and all training samples needs to be calculated during prediction, which may be inefficient for large-scale data sets.

2.2 classifiers comparison¶

Combined with the above code running results and the characteristics of classifiers, Bagging and AdaBoost perform better in all indicators, decision tree and random forest are similar and recall rate is low, K-nearest neighbor accuracy rate is good but recall rate is average.

2.3 conclusion¶

In terms of performance, Bagging>AdaBoost>K Nearest Neighbors> Decision Trees (Random Forest) According to this data set, Bagging is selected for comprehensive performance; AdaBoost is selected for adaptive adjustment and high recall; K nearest neighbors are selected for accuracy and nonlinear data; and decision trees are selected for interpretability. To solve the problem of large difference in classification number and overfitting, adjust the parameter decision tree and random forest; select the K value of K nearest neighbors by cross validation;Bagging and AdaBoost commutative classifiers or adjust the number of sets.
